Features

Everything you need for multi-agent orchestration

claude-swarm gives you a complete toolkit to manage, monitor, and coordinate multiple Claude Code agents working together.

🤖

Multi-Agent Orchestration

Run multiple Claude Code instances simultaneously. Each agent has its own role, model, skills, and permissions. An architect plans while a developer codes and a reviewer checks.

👥

Team Tasks (3-Phase)

Send team-wide tasks with :tt command. All agents collaborate through Planning → Executing → Synthesizing phases automatically.

💬

Inter-Agent Communication

Agents can send messages to each other. The Office view shows active meetings in real-time with animated cubicles and a meeting room.

🎨

3 Themes — Dark, Light, High Contrast

Switch themes instantly from the Settings tab. Dark mode for late nights, Light for bright environments, High Contrast for accessibility.

🌐

i18n — English & Turkish

All log messages, UI labels, and system messages are available in English and Turkish. Switch languages instantly from Settings.

📋

Structured Logs with Categories

Logs are categorized with icons: 🤖 Agent, 📋 Task, 👥 Team, 💬 Comm, ⚙ System, ❌ Error. Filter by verbosity level.

📝

Multi-line Input with History

Enter creates new lines, Ctrl+Enter submits. Input grows dynamically (3-10 lines). Up/Down navigates history.

⚙️

Persistent Settings

All preferences saved to ~/.claude-swarm/settings.toml. Language, theme, log verbosity, and more — all configurable.

🖥️

7 View Tabs

Dashboard, Agent Detail, Tasks, Logs, Office, Settings, Performance — each with a dedicated view.

📱

Telegram Integration

Control your swarm remotely via Telegram bot. Send tasks, check status, get notifications — all from your phone.

⏰

Scheduled Tasks

Schedule commands with :schedule 09:00 :t dev fix bug. Local time, 24h format.

📊

Performance View

Process resources, aggregate stats (cost/hr, success rate), per-agent token usage — all in real-time.

🔨

Build Verification

Auto-verify builds after agent completion. Auto-detect build system. Retry on failure.

Configuration

Simple TOML-based configuration

swarm.toml

Define your agents with roles, models, skills, and permissions.

# Swarm metadata
name = "my-project-swarm"
description = "Multi-agent dev team"

# Lead agent
[[agent]]
name = "architect"
role = "Lead Architect"
model = "opus"
skills = ["architecture", "planning"]
is_lead = true

# Worker agent
[[agent]]
name = "developer"
role = "Full-Stack Developer"
model = "sonnet"
skills = ["coding", "testing"]
permission_mode = "full-auto"

settings.toml

User preferences saved to ~/.claude-swarm/settings.toml

language = "en"        # en | tr
theme = "dark"          # dark | light | highcontrast
log_verbosity = "normal" # minimal | normal | detailed
terminal_app = "Terminal"
input_history_size = 50
meeting_timeout_secs = 10
auto_verify = false     # auto build check
telegram_enabled = false
telegram_bot_token = ""

Installation

Get started in 60 seconds

1

Prerequisites

Rust 1.70+ and Claude Code CLI installed and authenticated.

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
2

Clone & Build

git clone https://github.com/BirdyTheDev/claude-swarm.git
cd claude-swarm
cargo install --path .
3

Configure

cp swarm.toml.example swarm.toml
# Edit to define your agents
vim swarm.toml
4

Run

claude-swarm --config swarm.toml

# With an initial prompt:
claude-swarm --config swarm.toml --prompt "Analyze the codebase"

# With specific agents only:
claude-swarm --config swarm.toml --agents architect,developer

CLI Options

claude-swarm [OPTIONS]

  -c, --config <FILE>       Config file [default: swarm.toml]
  -p, --prompt <TEXT>       Initial prompt for lead agent
      --tick-rate <MS>      TUI refresh rate [default: 250]
      --agents <NAMES>      Comma-separated agent names
      --visible-terminals   Open each agent in a terminal
      --log-file <FILE>     Log file [default: claude-swarm.log]
      --log-level <LEVEL>   Log level [default: info]
